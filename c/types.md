<summary>
  Liggle- and Big- endian
</summary>

**Little-endian (LE)** - это порядок байтов **от младшего к старшему**.

```c
uint32_t ui32 = 0xABCDEF01;
```

Но в памяти будет расположен следующим образом

| Адрес | 0     | 1     | 2     | 3     |
|-------|-------|-------|-------|-------|
| Байт  | 0x01  | 0xEF  | 0xCD  | 0xAB  |

Он используется современными процессорами x86/x86-64

---

**Big-endian (BE)** - это порядок байтов **от старшего к младшему**.

```c
uint32_t ui32 = 0xABCDEF01;
```

| Адрес | 0     | 1     | 2     | 3     |
|-------|-------|-------|-------|-------|
| Байт  | 0xAB  | 0xCD  | 0xEF  | 0x01  |

Он используется:

- в сетевых протоколах (например, TCP/IP)
- во многих форматах файлов — PNG, FLV, JPEG
- в редких архитектурах процессоров

Модели данных и размеры типов
</summary>

<details>
<summary>
Модель LP32
</summary>

2 / 4 / 4 (int - 16 бит, long и pointers - 32 бита)
Используется для 32-разрядных систем в Win16 API.
</details>

<details>
<summary>
Модель ILP32
</summary>

4 / 4 / 4 (int, long и pointers - 32 бита)
Используется для 32-разрядных систем в Win32 API, Unix, Unix-подобных системах (Linux, macOS).
</details>

<details>
<summary>
Модель LLP64
</summary>

4 / 4 / 8 (int и long - 32 бита, pointers - 64 бита)
Используется для 64-разрядных систем ARM (AArch64) и x86-64 (x64) в Win32 API (современный Windows).
</details>

<details>
<summary>
Модель LP64
</summary>

4 / 8 / 8 (int - 32 бита, long и pointers - 64 бита)
Используется для 64-разрядных систем Unix и Unix-подобных системах (современный Linux, macOS).
</details>

<details>
<summary>
Тип float
</summary>

Гарантированный размер: 4 байта (32 бита)
Используется на всех архитектурах: x86, x86-64, ARM
</details>

<details>
<summary>
Тип double
</summary>

Гарантированный размер: 8 байт (64 бита)
Используется на всех архитектурах: x86, x86-64, ARM
</details>

<details>
<summary>
Тип long double
</summary>

Размер зависит от архитектуры:

- x86: 10 байт (80 бит)
- x86-64: 16 байт (128 бит)
- ARM: 8 байт (64 бита)
Гарантированный минимум: 8 байт (64 бита)

</details>

<details>
<summary>
Тип bool
</summary>

Размер: обычно 1 байт (8 бит)
Значение sizeof(bool) определяется реализацией и может отличаться от 1
</details>

<details>
<summary>
Проверка размеров типов
</summary>

Для определения размеров типов на вашей системе используйте оператор sizeof():
```c
printf("int: %zu байт\n", sizeof(int));
printf("long: %zu байт\n", sizeof(long));
printf("pointer: %zu байт\n", sizeof(void*));
printf("float: %zu байт\n", sizeof(float));
printf("double: %zu байт\n", sizeof(double));
printf("long double: %zu байт\n", sizeof(long double));
printf("bool: %zu байт\n", sizeof(bool));
```
</details>

<summary>
  Минимальные размеры типов (стандарт C)
</summary>

| Тип | Минимальный размер | Минимальный диапазон |
|-----|-------------------|---------------------|
| `char` | 1 байт (8 бит) | -127 до 127 или 0 до 255 |
| `signed char` | 1 байт (8 бит) | -127 до 127 |
| `unsigned char` | 1 байт (8 бит) | 0 до 255 |
| `short` | 2 байта (16 бит) | -32767 до 32767 |
| `unsigned short` | 2 байта (16 бит) | 0 до 65535 |
| `int` | 2 байта (16 бит) | -32767 до 32767 |
| `unsigned int` | 2 байта (16 бит) | 0 до 65535 |
| `long` | 4 байта (32 бита) | -2147483647 до 2147483647 |
| `unsigned long` | 4 байта (32 бита) | 0 до 4294967295 |
| `long long` | 8 байт (64 бита) | -9223372036854775807 до 9223372036854775807 |
| `unsigned long long` | 8 байт (64 бита) | 0 до 18446744073709551615 |

**Подсказка:** Чтобы определить максимальное значение для целого типа, используйте формулу:  
Для unsigned: `2^(биты) - 1`  
Для signed: `-2^(биты-1)` до `2^(биты-1) - 1`

<summary>
Числа с плавающей точкой (IEEE-754)
</summary>

| Тип | Размер | Точность | Диапазон (приблизительно) | Мантисса | Экспонента |
|-----|--------|----------|---------------------------|----------|------------|
| `float` | 4 байта (32 бита) | ~7 десятичных цифр | ±1.2×10⁻³⁸ до ±3.4×10³⁸ | 23 бита | 8 бит |
| `double` | 8 байт (64 бита) | ~15-16 десятичных цифр | ±2.2×10⁻³⁰⁸ до ±1.8×10³⁰⁸ | 52 бита | 11 бит |
| `long double` | 10-16 байт | ~18-34 десятичных цифр | Зависит от реализации | 64-112 бит | 15-17 бит |

**Структура IEEE-754:**
```
[знак][экспонента][мантисса]
```

**Особенности:**
- **Знак**: 1 бит (0 = положительный, 1 = отрицательный)
- **Экспонента**: хранится со смещением (bias)
- **Мантисса**: нормализованное число 1.xxxxx в двоичной системе
- **Специальные значения**: NaN, ±Infinity, ±0

**Пример для float (32 бита):**
```
0 10000000 10010010000111111011011
↑ ↑        ↑
│ │        └── мантисса (23 бита)
│ └─────────── экспонента (8 бит)  
└───────────── знак (1 бит)
```
